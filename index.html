History API / Routing
Browsers no longer handle routing.

SPAs to the rescue.

View as a function of state.
The DOM Window object provides access to the browser's session history through the history object
Instance methods of History

							window.history.back() // backward through history
							window.history.forward() // forward throug history
							window.history.go(-1) // back one page
							window.history.go(1) // forward one page

history.pushState()


							pushState(state, unused)
                            pushState(state, unused, url)

Method pushState() adds an entry to the browser's session history stack.

history.replaceState()


							replaceState(state, unused)
                            replaceState(state, unused, url)
Instance properties of History
history.length

//read-only property returns an integer representing the number of elements in the session history, including the currently loaded page

history.scrollRestoration

// allows web applications to explicitly set default scroll restoration behavior on history navigation.

history.state

// property returns a value representing the state at the top of the history stack
